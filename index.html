Megaline — Descriptive & Inferential Statistics

Statistical analysis of telecom customer behavior to identify revenue differences between two prepaid mobile plans.

Project Overview

This project applies descriptive and inferential statistics to explore customer usage patterns and revenue generation. The objective is to understand which prepaid plan—Surf or Ultimate—brings higher earnings for Megaline.

Problem: The commercial department needs to determine which plan generates higher revenue to optimize advertising budget allocation.

Technologies & Tools: Python (pandas, numpy, scipy, matplotlib, seaborn, math), Jupyter Notebook, VS Code

Exploratory Data Analysis

  • Average call duration by plan and month
  • Monthly minutes per plan
  • Average messages per plan
  • MB consumed per plan
  • Average monthly revenue per plan

Conclusion

  • No significant differences were found in call duration, messages sent, or monthly data usage between the two plans.
  • A t-test revealed a significant difference in average revenue: the Ultimate plan generates more income on average.
  • Surf plan shows higher revenue volatility due to users generating extra charges through exceeding usage.
  • Ultimate’s large minute allowance may be excessive, as most users consume less than what is offered.
  • Data consumption in Surf increases in the last months of the year, suggesting opportunities for data-focused commercial strategies.
  • Megaline should target atypical high-consumption users, who generate the highest revenue, either within existing plans or with a redesigned offer.

ICE — Video Game Sales Analysis

Complete exploratory analysis of a real dataset containing video game sales. The goal is to identify patterns, regional differences, and success factors that can guide commercial and advertising strategies for the upcoming year.

Project Overview

Technologies & Tools: Python (pandas, numpy, scipy, matplotlib, seaborn, math), Jupyter Notebook, VS Code

Actions Performed:

  • Prepared and cleaned the dataset, converting data types when required.
  • Handled missing values by imputing or removing them based on impact.
  • Created additional features such as total sales.
  • Performed deep exploratory analysis: platform lifecycle, global sales, genre distribution, critic/user scores, etc.
  • Conducted regional analysis (NA, EU, JP, other regions).
  • Performed hypothesis testing:
    • User ratings for Xbox One and PC are the same.
    • User ratings for Action and Sports genres are different.

Exploratory Data Analysis

  • Games released per year
  • Sales by platform
  • Yearly video game sales
  • Distribution of global sales by platform
  • Sales distribution by genre

Conclusion

  • The video game market shows strong volatility and dynamism in recent years.
  • Although average platform sales are similar, outliers from highly popular games create large differences.
  • Platform maturity varies drastically, making it difficult to apply uniform commercial strategies.
  • Strategies must focus on platforms with strong sales in recent years, since formerly dominant ones are declining rapidly.
  • Regional trends differ significantly — especially between Japan and Western markets (EU/NA).
  • ESRB ratings have limited impact on sales overall but remain relevant for broad-audience titles.

Zuber — Exploratory Analysis of Ride-Sharing Trips

Analysis of rideshare data for Zuber, a new company launching in Chicago. The goal is to uncover passenger preferences and understand how external factors—particularly weather—affect trip behavior and duration.

Project Overview

Technologies & Tools: Python (pandas, numpy, scipy, matplotlib, seaborn, math), Jupyter Notebook, VS Code

Exploratory Data Analysis

  • Number of trips by company
  • Average trips by drop-off neighborhood (November)
  • Trip duration based on weather conditions

Conclusion

  • A hypothesis test showed a statistically significant difference in mean trip duration: Saturday rainy days affect rides from Loop to O'Hare, increasing trip duration.
  • Trip duration has noticeable outliers under both good and bad weather conditions.
  • The interquartile range is similar for both weather types, but the median duration is higher on rainy days.
  • Good-weather days display more extreme outliers, suggesting variability in traffic and demand patterns.

Megaline — Machine Learning Plan Recommendation

Development of a machine learning model capable of recommending the most suitable phone plan (Smart or Ultra) based on real customer usage behavior.

Project Overview

Megaline noticed that many customers still use legacy plans and want to transition them to more efficient options. The goal of this project was to create a predictive model that analyzes customer activity—minutes used, messages sent, and data consumption—and recommends the plan that best fits each customer profile.

Technologies & Tools

Python (pandas, scikit-learn), Jupyter Notebook, VS Code

What Was Done

  • Cleaned and prepared the dataset.
  • Split the data into training, validation, and test segments.
  • Trained several machine learning models:
    • Decision Tree
    • Random Forest
    • Logistic Regression
  • Compared model performance using accuracy.
  • Selected the best model for deployment.

Results

  • The best-performing model was Random Forest, achieving an accuracy of 81%.
  • This performance exceeds the 75% target accuracy required by Megaline.
  • The model generalizes well and shows no signs of overfitting.
  • The predictions are reliable enough to be used for customer plan recommendations.

Conclusion

  • This machine learning model can support Megaline in migrating customers to the Smart or Ultra plan based on their real usage patterns.
  • It helps improve customer experience and optimize commercial strategy.
  • The project demonstrates the full workflow of implementing a supervised classification model: data preparation, model selection, tuning, and evaluation.

BetaBank Churn Prediction

This project aims to predict which BetaBank customers are likely to leave. Due to significant class imbalance, multiple strategies were compared: class_weight, SMOTE, and undersampling.

Model Results

Model F1 (Test) AUC-ROC (Test)
Base (LR) 0.284 0.77
LR (class_weight='balanced') 0.498 0.777
Random Forest + SMOTE 0.620 0.859
XGBoost + UnderSampling 0.592 0.869

Visualizations

The recommended model is Random Forest + SMOTE, achieving the best balance between F1 and AUC-ROC. Regular monitoring of metrics is suggested to detect potential model drift.
